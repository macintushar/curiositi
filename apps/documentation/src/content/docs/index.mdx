## Quick Start

Get Curiositi running in minutes:

```bash
# Clone the repository
git clone https://github.com/macintushar/curiositi
cd curiositi

# Install dependencies
bun install

# Set up environment files
cp ./apps/server/.env.example ./apps/server/.env
cp ./apps/web/.env.example ./apps/web/.env

# Run with Docker Compose (includes PostgreSQL + pgvector)
docker compose up -d

# Or run migrations manually if using local Postgres
bun run db:migrate

# Start development servers
bun run dev
```

Access at: **http://localhost:3040** (Web UI) | **http://localhost:3030** (API)

## What is Curiositi?

Curiositi is a **self-hostable AI knowledge workspace** that combines:

- **Document Intelligence**: Upload PDFs, Office documents, and text files
- **Vector Search**: Semantic search powered by OpenAI embeddings and pgvector
- **RAG Agent**: AI agents that search your documents and the web in parallel
- **Streaming Chat**: Real-time responses with tool calling and source citations
- **Secure by Design**: better-auth with per-user API keys, session management
- **Multi-Provider**: OpenAI, Anthropic, and OpenRouter support

## **Tech Stack:**

- **Frontend**: Next.js 15, React 19, TanStack Query, Zustand, Tailwind v4
- **Backend**: Hono, Bun runtime, Drizzle ORM, better-auth
- **Database**: PostgreSQL 16+ with pgvector extension
- **AI**: Vercel AI SDK, LangChain (document processing)
- **Search**: Vector similarity (cosine distance), Firecrawl web search
