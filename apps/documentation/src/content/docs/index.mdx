## Quick Start

Get Curiositi running in minutes:

```bash
# Clone the repository
git clone https://github.com/macintushar/curiositi
cd curiositi

# Install dependencies
bun install

# Set up environment files
cp ./apps/server/.env.example ./apps/server/.env
cp ./apps/web/.env.example ./apps/web/.env

# Run with Docker Compose (includes PostgreSQL + pgvector)
docker compose up -d

# Or run migrations manually if using local Postgres
bun run db:migrate

# Start development servers
bun run dev
```

Access at: **http://localhost:3040** (Web UI) | **http://localhost:3030** (API)

## What is Curiositi?

Curiositi is a **self-hostable AI knowledge workspace** that combines:

- **Document Intelligence**: Upload PDFs, Office documents, and text files
- **Vector Search**: Semantic search powered by OpenAI embeddings and pgvector
- **RAG Agent**: AI agents that search your documents and the web in parallel
- **Streaming Chat**: Real-time responses with tool calling and source citations
- **Secure by Design**: better-auth with per-user API keys, session management
- **Multi-Provider**: OpenAI, Anthropic, and OpenRouter support

## Architecture Overview

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│  Next.js 15  │  HTTP   │   Hono API   │  SQL    │  PostgreSQL  │
│   Web UI     │ ◄─────► │   + Bun      │ ◄─────► │  + pgvector  │
└──────────────┘         └──────────────┘         └──────────────┘
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼
              ┌───────────┐           ┌───────────┐
              │  OpenAI   │           │ Firecrawl │
              │ Anthropic │           │ (Search)  │
              │OpenRouter │           └───────────┘
              └───────────┘
```

**Tech Stack:**
- **Frontend**: Next.js 15, React 19, TanStack Query, Zustand, Tailwind v4
- **Backend**: Hono, Bun runtime, Drizzle ORM, better-auth
- **Database**: PostgreSQL 16+ with pgvector extension
- **AI**: Vercel AI SDK, LangChain (document processing)
- **Search**: Vector similarity (cosine distance), Firecrawl web search

## Key Concepts

### Spaces
Isolated containers for organizing documents. Each space has:
- Unique vector index for scoped retrieval
- Independent file management
- Optional descriptions for better context

### Threads
Conversation threads that maintain chat history:
- Can be associated with specific spaces
- Store complete message history with tool calls
- Track token usage and model metadata

### Search Agent
The core AI component that:
1. **Analyzes** your question and conversation context
2. **Plans** optimal document and web search queries
3. **Executes** searches in parallel across sources
4. **Synthesizes** comprehensive answers with citations
5. **Suggests** relevant follow-up questions

### Vector Embeddings
Documents are split into chunks and embedded using OpenAI's `text-embedding-3-small` (1024 dimensions):
- Stored in PostgreSQL with pgvector extension
- Searched using cosine similarity (threshold > 0.3)
- Retrieved in parallel for multiple queries
